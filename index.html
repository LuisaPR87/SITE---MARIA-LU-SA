Controle de Versão - GIT

Primeiramente, antes de tentar entender o que é GitHub, é necessário saber o conceito de con-
trole de versão e Git. Então, vamos lá!

O que é Controle de Versão?

É uma maneira de gerenciar as mudanças ocorridas em um código, ou seja, pode-se dizer que
é um sistema que registra alterações em um arquivo ou grupo de arquivos ao longo do tempo,
para assim, você se lembrar das versões e recuperar elas. Existem 3 tipos de controle de ver-
são, são eles: Centralizados, Distribuidos e Locais.

- Centralizados: desenvolvedores trabalham juntos em um único projeto, tendo apenas um re-
positório .

- Distribuidos: Os desenvolvedores do projeto podem trabalhar juntos ao mesmo tempo, com is-
so, ele é recomendado para projetos que possuem mais de um desenvolvedor onde, todo o reposi-
tório é copiado.

Locais: Ele armazena todos as alterações na máquina.

O que é Git?

Git é um controle de versões, desenvolvido por Linus Torvalds. Ele é usado para "controlar as versões".
No Git, são armazenados snapshots, ou seja, é como se fosse uma foto da situação do sistema naquela
hora. Ele registra três estados, são eles: consolidade (commited), modificado(modified) e preparado
(staged).

> Consolidade: Ficam armazenados na base local.
> Modificado: O arquivos sofre modificações, porém ainda não teve commit.
> Praparado: Arquivos que irão fazer parte do pró-
ximo snapshot.
