Controle de Versão - GIT

Primeiramente, antes de tentar entender o que é GitHub, é necessário saber o conceito de con-
trole de versão e Git. Então, vamos lá!

O que é Controle de Versão?

É uma maneira de gerenciar as mudanças ocorridas em um código, ou seja, pode-se dizer que
é um sistema que registra alterações em um arquivo ou grupo de arquivos ao longo do tempo,
para assim, você se lembrar das versões e recuperar elas.

Existem 3 tipos de controle de versão, são eles: centralizados, distribuidos e locais

- Centralizados: desenvolvedores trabalham juntos em um único projeto, tendo apenas um repositório .

- Distribuidos: Os desenvolvedores do projeto podem trabalhar juntos ao mesmo tempo, com isso, ele é recomendado para projetos que possuem mais de um desenvolvedor onde, todo o repositório é copiado.

- Locais: Ele armazena todos as alterações na máquina.

O que é Git?

Git é um controle de versões, desenvolvido por Linus Torvalds. Ele é usado
para "controlar as versões".
